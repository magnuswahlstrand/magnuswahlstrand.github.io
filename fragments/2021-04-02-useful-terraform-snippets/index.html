Useful Terraform snippets

A while back I was introduced to Terraform by a colleague. Since then, I have started using it more and more both at work and for my own projects.

Terraform is a declarative tool for managing infrastructure. You specify what resources and resource state you want, in configuration files, and Terraform gets it to that state. I have used some of HashiCorp’s tools in the past (Consul, Vault, …) and they all have great user experience and good documentation, and Terraform is no exception.

Snippets

.gitignore

Before you start, make sure to use a good .gitignore configuration to avoid committing secrets and sensitive information. Here is Github’s Terraform .gitignore.

Terraform S3 backend

For production systems it is better to store state centrally, rather than locally. I have used S3 for this. More information here: Terraform S3 Backend.

terraform {
  backend "s3" {
    bucket = "heroku-terraform-backend-v1"
    key = "heroku.tfstate"
    region = "eu-north-1"
    profile = "terraform-s3"
  }
}

Heroku

terraform {
 ...
  # Include the Heroku provider
  required_providers {
    heroku = {
      source = "heroku/heroku"
      version = "3.2.0"
    }
  }
}

# Configure the Heroku provider
// https://www.terraform.io/docs/language/values/variables.html#variable-definitions-tfvars-files
provider "heroku" {
  email = var.heroku_email
  api_key = var.heroku_api_key
}

Loading JSON

From S3

// https://registry.terraform.io/providers/hashicorp/aws/latest/docs/data-sources/s3_bucket_object
data "aws_s3_bucket_object" "sensitive_vars" {
    bucket = "my-very-sensitive-data-bucket"
    key    = "data.json"
}

locals {
    sensitive = jsondecode(data.aws_s3_bucket_object.sensitive_vars.body)
}

From GCP storage

data "google_storage_bucket_object_content" "sensitive_gcp_vars" {
  bucket = "my-very-sensitive-data-bucket-on-gcp"
  name   = "folder/butterfly01.json"
}


locals {
  sensitive_gcp = jsondecode(data.google_storage_bucket_object_content.sensitive_gcp_vars.content)
}

Using local variables

resource "heroku_app" "default" {
    name = "my-cool-terraform-app"
    region = "eu"

    config_vars = {
        FOOBAR = "barfoo2"
    }

    sensitive_config_vars = {
        USERNAME = local.sensitive.username,
        PASSWORD = local.sensitive.password,
    }

    buildpacks = [
        "heroku/go"
    ]
}
